------------------------------------------------
17. AI AGENT EXECUTION RULES
------------------------------------------------

Before making any change:

1. Audit the existing implementation.

2. Explain the root cause.

3. Identify affected files.

4. List risks.

5. Present an implementation plan.

6. Wait until the audit is complete before making structural changes.

7. Prefer extending existing components over creating new ones.

8. Preserve backward compatibility.

9. Do not rewrite working code without measurable improvement.

10. Never remove functionality unless explicitly instructed.

11. Never introduce breaking changes.

12. Keep components reusable.

13. Respect the existing design system.

14. Respect architecture established in previous phases.

15. Validate the completed work against the Enterprise Quality Gate before considering the task complete.
